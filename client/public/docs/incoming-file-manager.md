# Incoming File Manager

The **Incoming File Manager** provides a **centralized view of files generated by the scanning process**. Each entry represents the result of a validation check performed before a file is allowed to move into the production vault.

This page helps ensure that incoming files meet required standards, preventing errors, duplication, and untracked changes prior to release.

![Incoming File Manager](./images/ifm.png "width=800 nozoom")  
**Access:** Top Menu → **Utilities**, Left Menu → **Incoming File Manager**. <br>

## Key Columns

The Incoming File Manager includes key indicators that help users quickly assess scan results.

### Scan Status

**Scan Status** reflects the outcome of the validation:

- **Development** – File is still under development and not ready for release
- **Exceptions** – Issues were detected that require review or correction
- **Modified** – Differences were found compared to the existing reference
- **Identical** – No differences were detected; the file matches the existing reference

### Compare Type

**Compare Type** indicates how the file was evaluated:

- **CNC Scan** – File was scanned and compared against existing CNC references
- **Incoming Folder** – File was validated as part of an incoming folder scan

## Typical Workflow

1. Review the scan results listed in the Incoming File Manager
2. Check the **Scan Status** to identify files requiring action
3. Use the **Compare Type** to understand how the file was evaluated
4. Address exceptions or modifications before releasing files to production

This structured review process ensures that only validated and traceable files are introduced into the production environment.
